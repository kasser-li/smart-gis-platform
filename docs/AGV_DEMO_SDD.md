## 一、需求分析

### 1.1 背景
在现有 GIS 解析图平台基础上，新增 AGV（自动导引车）业务演示模块，用于：
- 展示 AGV 调度系统核心能力
- 客户演示与方案验证
- 技术可行性探索

### 1.2 目标用户
- 潜在客户（智慧工厂、仓储物流）
- 内部技术团队
- 方案售前支持

### 1.3 核心需求

| 需求 | 描述 | 优先级 |
|------|------|--------|
| AGV 监控 | 实时显示 AGV 位置、状态、电量 | P0 |
| 任务管理 | 创建、分配、跟踪搬运任务 | P0 |
| 路径规划 | 可视化展示 AGV 行驶路线 | P0 |
| 充电管理 | 充电站管理、低电量自动充电 | P1 |
| 数据看板 | 运营数据统计展示 | P1 |
| 模拟演示 | 可交互的 AGV 运行模拟 | P1 |

## 三、数据设计

### 3.1 AGV 数据结构

```typescript
interface AGV {
  id: string           // 编号，如 "001"
  status: string       // 状态：工作中/空闲/充电中/离线/故障
  battery: number      // 电量 0-100
  load: number         // 当前负载 (kg)
  maxLoad: number      // 最大负载 (kg)
  speed: number        // 当前速度 (m/s)
  position: {          // 位置坐标
    lat: number
    lng: number
  }
  currentTask: string | null  // 当前任务 ID
  runTime: number      // 运行时长 (分钟)
}
```

### 3.2 任务数据结构

```typescript
interface Task {
  id: string           // 任务 ID，如 "T001"
  type: string         // 类型：搬运/分拣/充电/巡检
  status: string       // 状态：等待/进行中/已完成/已取消
  agvId: string        // 分配的 AGV ID
  from: {              // 起点
    id: string
    name: string
  } | null
  to: {                // 终点
    id: string
    name: string
  } | null
  priority: number     // 优先级 1-低 2-中 3-高
}
```

### 3.3 工作站数据结构

```typescript
interface Waypoint {
  id: string           // 编号，如 "W1"
  name: string         // 名称，如 "原料仓库 A"
  position: {
    lat: number
    lng: number
  }
}
```

### 3.4 充电站数据结构

```typescript
interface ChargingStation {
  id: string           // 编号，如 "C1"
  name: string         // 名称，如 "1 号充电站"
  position: {
    lat: number
    lng: number
  }
  status: string       // 状态：空闲/占用
}
```

## 四、功能设计

### 4.1 AGV 监控

**功能描述**: 实时展示所有 AGV 的状态和位置

**交互流程**:
1. 页面加载时获取 AGV 列表
2. 在地图上显示 AGV 标记
3. 左侧表格显示详细信息
4. 点击表格行定位到对应 AGV
5. 点击 AGV 标记显示详情弹窗

**状态定义**:
| 状态 | 颜色 | 说明 |
|------|------|------|
| 工作中 | 蓝色 | 正在执行任务 |
| 空闲 | 绿色 | 可用状态 |
| 充电中 | 橙色 | 正在充电 |
| 离线 | 灰色 |  disconnected |
| 故障 | 红色 | 需要维修 |

### 4.2 任务管理

**功能描述**: 创建、分配、跟踪搬运任务

**交互流程**:
1. 点击"新建任务"按钮
2. 填写任务表单（类型、AGV、起点、终点、优先级）
3. 系统自动分配或手动选择 AGV
4. 任务进入等待队列
5. AGV 开始执行后状态变为"进行中"
6. 完成任务后状态变为"已完成"

**任务类型**:
- 搬运：从 A 点搬运物料到 B 点
- 分拣：在指定区域进行分拣作业
- 充电：前往充电站充电
- 巡检：按指定路线巡检

**分配策略**:
- 自动分配：选择电量>20% 且空闲的 AGV
- 手动指定：从可用 AGV 列表选择

### 4.3 路径规划

**功能描述**: 可视化展示 AGV 行驶路线

**实现方案**:
- 简化版：直线连接起点和终点（虚线）
- 起点标记：绿色圆点
- 终点标记：红色圆点
- 路径颜色：蓝色 (#409EFF)

**扩展方向** (未来):
- A*算法避障路径规划
- 多 AGV 路径冲突检测
- 动态路径重规划

### 4.4 充电管理

**功能描述**: 管理 AGV 充电行为

**自动充电逻辑**:
```
if (AGV.battery < 15% && AGV.status === '工作中') {
  AGV.status = '充电中'
  AGV.currentTask = null
  // 导航到最近空闲充电站
}

if (AGV.status === '充电中') {
  AGV.battery += 0.5%/秒
  if (AGV.battery >= 100%) {
    AGV.status = '空闲'
  }
}
```

### 4.5 模拟演示

**功能描述**: 模拟 AGV 真实运行场景

**模拟行为**:
- AGV 位置每秒随机偏移（模拟移动）
- 工作时电量持续消耗（-0.1%/秒）
- 充电时电量恢复（+0.5%/秒）
- 低电量自动触发充电
- 运行时长累计

**控制按钮**:
- ▶ 开始模拟：启动定时更新
- ⏸️ 暂停模拟：停止更新
- ↻ 重置：恢复初始状态

### 4.6 数据看板

**功能描述**: 展示关键运营指标

**指标定义**:
| 指标 | 计算方式 |
|------|----------|
| 在线 AGV | status ≠ '离线' 的数量 |
| 进行中任务 | status = '进行中' 的数量 |
| 平均电量 | SUM(battery) / AGV 总数 |
| 完成任务 | status = '已完成' 的数量 |

## 七、风险评估

### 7.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|----------|
